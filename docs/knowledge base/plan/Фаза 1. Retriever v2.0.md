## Контекст

Фаза 1 реализована: retriever/ работает (2651 чанк проиндексирован, гибридный поиск отдаёт результаты). Проблема: **BM25 плохо работает на natural language запросах** — шумит, даёт низкую точность. RRF-скоры (max ~0.5) подтверждают, что sparse-ветка часто не помогает.

Идея пользователя: **разделить запросы** для dense и sparse веток. LLM-агент на первой ноде анализирует запрос и извлекает «жёсткие термины» (даты, аббревиатуры, коды, имена собственные). Эти термины передаются отдельным параметром `bm25_terms` в инструмент поиска. BM25 ищет только по ним — точное совпадение, а не шум.

Также нужен **опциональный реранкер** (cross-encoder) после fetch_k кандидатов.

---

## Анализ идеи и корректировки

**Что хорошо:**

- BM25 заточен под exact match — отдельные термины идеально ему подходят
- LLM лучше знает, какие слова «жёсткие» (ШР, 30/2024, 15.03.2025, HITL)
- Dense-only без BM25-шума может дать лучшие результаты на обычных вопросах

**Корректировки:**

1. `bm25_terms=None` → **dense-only** (пропускаем BM25 целиком, не используем оригинальный запрос)
2. Пустую строку `""` трактуем как None (guard через `.strip()`)
3. `limit` адаптируется: с реранкером забираем `fetch_k` кандидатов (больше материала для пересортировки), без — сразу `max_chunks`
4. Реранкер использует **pure re-sort** (score реранкера заменяет RRF/cosine score). Взвешенная комбинация (`reranker_weight`) хрупкая — шкалы разные (RRF 0–0.5, cosine 0–1, cross-encoder -10–+10). Pure re-sort проще и стандартнее

### 3.1. Разделение `_embed_query` → `_embed_dense` + `_embed_sparse`

Старая функция `_embed_query` всегда вычисляла оба вектора. Теперь — раздельно:

python

```python
def _embed_dense(query: str) -> list[float]:
    """Эмбеддит запрос → dense-вектор (E5-large, автопрефикс "query: ")."""
    store = get_qdrant_store()
    return store.embeddings.embed_query(query)

def _embed_sparse(text: str) -> qmodels.SparseVector:
    """Эмбеддит термины → sparse-вектор (BM25). Принимает жёсткие термины, не запрос."""
    store = get_qdrant_store()
    sparse_raw = store._sparse_embeddings.embed_query(text)
    return qmodels.SparseVector(indices=sparse_raw.indices, values=sparse_raw.values)
```

### 3.2. Синглтон реранкера

python

```python
from fastembed.rerank.cross_encoder import TextCrossEncoder

_reranker: TextCrossEncoder | None = None

def _get_reranker() -> TextCrossEncoder:
    """Ленивый синглтон кросс-энкодера. Загружается только при use_reranking=True."""
    global _reranker
    if _reranker is None:
        cfg = get_config().search
        _reranker = TextCrossEncoder(cfg.reranker_model)
    return _reranker
```

> Почему `fastembed.TextCrossEncoder`?
> 
> - ONNX-оптимизация → быстрее на CPU чем PyTorch (sentence-transformers)
> - `fastembed` уже установлен (зависимость langchain-qdrant)
> - `BAAI/bge-reranker-v2-m3` поддерживается, мультиязычный (100+ языков, русский)
> - `bge-reranker-base` (PyTorch) был слишком медленным в другом проекте — ONNX должен быть заметно быстрее

### 3.3. Функция `_rerank`

python

```python
def _rerank(query: str, results: list[SearchResult]) -> list[SearchResult]:
    """
    Pure re-sort: кросс-энкодер заменяет score целиком.
    Кросс-энкодер видит (запрос, документ) вместе → его оценка точнее bi-encoder.
    Фильтрует по reranker_score_threshold.
    """
    cfg = get_config().search
    reranker = _get_reranker()
    documents = [r.content for r in results]

    # TextCrossEncoder.rerank() возвращает list[TextCrossEncoderRerankResult]
    # с полями .score и .doc_index
    rerank_results = list(reranker.rerank(query, documents, len(documents)))

    reranked = []
    for rr in rerank_results:
        original = results[rr.doc_index]
        if rr.score >= cfg.reranker_score_threshold:
            reranked.append(
                SearchResult(content=original.content, metadata=original.metadata, score=rr.score)
            )

    # rerank() уже отсортирован по score (убывание)
    return reranked[:cfg.max_chunks]
```

> **API fastembed**: `reranker.rerank(query, documents, top_k)` → `list[TextCrossEncoderRerankResult]` с `.score` (float) и `.doc_index` (int). Результат уже отсортирован по убыванию score.

### 3.4. Главная функция `search`

python

```python
def search(query: str, bm25_terms: str | None = None) -> list[SearchResult]:
    cfg = get_config().search
    store = get_qdrant_store()
    dense_vec = _embed_dense(query)

    if bm25_terms is not None and bm25_terms.strip():
        # --- гибридный: dense(query) + BM25(bm25_terms) + RRF ---
        sparse_vec = _embed_sparse(bm25_terms.strip())
        points = store.client.query_points(
            collection_name=store.collection_name,
            prefetch=[
                qmodels.Prefetch(
                    query=dense_vec, using=_DENSE, limit=cfg.fetch_k,
                    score_threshold=cfg.dense_score_threshold or None,
                ),
                qmodels.Prefetch(
                    query=sparse_vec, using=_SPARSE, limit=cfg.fetch_k,
                    score_threshold=cfg.sparse_score_threshold or None,
                ),
            ],
            query=qmodels.FusionQuery(fusion=qmodels.Fusion.RRF),
            limit=cfg.fetch_k,   # всегда fetch_k — обрежем позже
            with_payload=True,
        )
    else:
        # --- dense-only: семантический поиск ---
        points = store.client.query_points(
            collection_name=store.collection_name,
            query=dense_vec,
            using=_DENSE,
            limit=cfg.fetch_k,
            score_threshold=cfg.dense_score_threshold or None,
            with_payload=True,
        )

    results = [r for p in points.points if (r := _point_to_result(p)) is not None]

    if cfg.use_reranking and results:
        # реранкер: пересортировка + фильтрация по reranker_score_threshold
        results = _rerank(query, results)
    else:
        results = results[:cfg.max_chunks]

    return results
```

**Логика порогов (3 уровня фильтрации):**

1. **dense_score_threshold** — применяется на стороне Qdrant в `Prefetch` (dense-ветка) или в top-level query (dense-only). Фильтрует по cosine similarity. `0.0` = не фильтруем → `None` для Qdrant API.
2. **sparse_score_threshold** — применяется в `Prefetch` (sparse-ветка). Фильтрует BM25-кандидатов до RRF. `0.0` = `None`.
3. **reranker_score_threshold** — применяется после кросс-энкодера в `_rerank()`. Фильтрует по score реранкера.

`fetch_k` кандидатов всегда забираем из Qdrant, `max_chunks` — жёсткий лимит на выход.

> **`or None`**: Qdrant API трактует `score_threshold=None` как «не фильтровать». При `0.0` — зависит от реализации (может отфильтровать отрицательные). `or None` при дефолте 0.0 даёт чистое «без фильтра».
