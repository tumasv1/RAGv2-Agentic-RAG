## ADR-0009: Web UI — дизайн и компоновка интерфейса

**Статус:** принято

**Дата:** 23.04.26

---

## Контекст

FastAPI-интерфейс получил базовый HTML/CSS в рамках Фазы 4. Он работал, но был типовым: тёмная шапка, белый фон, стандартные кнопки Bootstrap-стиля. После добавления истории сессий (ADR-0008) появилась боковая панель и стало очевидно, что нужен осознанный дизайн, а не набор дефолтных стилей.

Интерфейс должен выглядеть профессионально и лаконично, работать на мобайле и десктопе, не отвлекать от содержимого.

---

## Варианты

**Вариант А:** Оставить тёмную тему (`#111827` шапка, `#f7f7f8` фон)
- Плюсы: привычная для dev-инструментов
- Минусы: тяжело, нет связи с бежевым тоном Obsidian-заметок

**Вариант Б:** Тема «Minimal 2026» — тёплый off-white, чёрный текст, hairline-границы
- Плюсы: воздушно, читаемо, перекликается с тоном бумаги/блокнота; без теней и лишних украшений
- Минусы: минималистичность требует аккуратности в деталях (типографика, отступы)

**Вариант В:** UI-фреймворк (Tailwind, Bootstrap)
- Плюсы: быстро, готовые компоненты
- Минусы: избыточный бандл для статичного сервера, потеря контроля над деталями

---

## Решение

Выбираем **Вариант Б** — тема Minimal 2026, реализованная ручным CSS без зависимостей.

### Цветовая палитра

| Токен | Значение | Где |
|-------|----------|-----|
| body / topbar bg | `#faf9f6` | тёплый off-white |
| sidebar bg | `#f4f2ec` | чуть теплее основного |
| разделители | `#ebe8df` | hairline |
| основной текст | `#0f0f0f` | почти чёрный |
| приглушённый текст | `#9a9888` | подписи, мета |
| пузырь пользователя | `#0f0f0f` фон, `#faf9f6` текст | инвертированный |
| пузырь агента | прозрачный фон, `1px solid #0f0f0f` | рамка hairline |

### Компоновка (L-образный layout)

На десктопе (≥ 1024 px):
- Sidebar фиксированный (`position: fixed; top: 0; bottom: 0`), ширина `--sidebar-w: 280px`
- Бренд (лого + «Agentic RAG») — только в sidebar, не в топбаре
- Топбар смещён `padding-left: var(--sidebar-w)` — кнопки Chat/Debug/Retriever/Admin начинаются над чатом
- `main-area` имеет `margin-left: var(--sidebar-w)` — не перекрывается sidebar'ом

На мобайле (< 1024 px):
- Sidebar скрыт (`transform: translateX(-100%)`), выезжает по гамбургеру
- Гамбургер без окантовки, opacity-hover
- Бренд (лого + название) появляется в топбаре
- Nav-ссылки в топбаре скрыты — места мало

### Typing indicator

Вместо текста «Обрабатываю…» (который смещал поле ввода) — три несинхронно мигающие точки внутри `#chat-history`, на месте будущего ответа агента. Паттерн DeepSeek: `@keyframes typing-dot` с opacity + translateY, задержки 0 / 0.2s / 0.4s.

### Прочие решения

- Кнопка «+ New chat» и список сессий только в sidebar; кнопка закрытия sidebar удалена (закрывается по клику на overlay / Escape)
- Thread ID скрыт из UI (был виден как «Session: UUID» — занимал место)
- Граница над полем ввода убрана (была избыточной)
- Тема задаётся классом `body.theme-minimal`; все стили изолированы в `theme-minimal.css` — можно добавить другую тему не ломая базовые стили

---

## Последствия

- CSS без фреймворков → полный контроль, нет версионных конфликтов
- Два файла для стилей: `app.css` (структура, layout, компоненты) + `theme-minimal.css` (цвета, оформление темы) — чёткое разделение
- Для новой темы достаточно добавить `theme-X.css` и переключить класс `body`
- При изменении `--sidebar-w` (CSS-переменная) layout перестраивается автоматически (sidebar, margin, padding-left топбара)
- На очень узких десктопах (1024–1100 px) nav-ссылки могут не помещаться — не критично для домашнего сервера
