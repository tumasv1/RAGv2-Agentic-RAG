## ADR-0011: PWA-совместимость веб-интерфейса

**Статус:** принято

**Дата:** 27.04.26

---

## Контекст

Приложение запускается на домашнем сервере и доступно в локальной сети. Основной сценарий использования — мобильный телефон в одной Wi-Fi-сети с сервером. Открывать браузер, вводить IP и порт каждый раз неудобно. Нужен способ запускать приложение с домашнего экрана, как нативное приложение.

---

## Варианты

**Вариант А: Progressive Web App (PWA)**
- Плюсы: работает в браузере без сборки, «Добавить на экран Home» встроено в Chrome/Safari, автономный режим (standalone) убирает браузерный UI, иконка на рабочем столе
- Минусы: требует HTTPS для service worker и установки (обход — ngrok или Tailscale); без HTTPS PWA открывается, но не устанавливается через Chrome

**Вариант Б: Нативное мобильное приложение (React Native / Flutter)**
- Плюсы: полный контроль над UI, push-уведомления
- Минусы: отдельная кодовая база, сборка, публикация в магазин — избыточно для домашнего инструмента

**Вариант В: Telegram-бот (уже запланирован в роадмапе)**
- Плюсы: уже есть кнопка в телефоне, уведомления, не нужен браузер
- Минусы: параллельная задача, не заменяет полноценный веб-интерфейс с историей сессий и debug-режимом

---

## Решение

Выбираем **Вариант А (PWA)**, потому что:
- Нулевые накладные расходы — всё делается в рамках существующего FastAPI-приложения
- Пользователь уже пользуется веб-интерфейсом на телефоне через Chrome
- Telegram-бот — отдельная фаза роадмапа, PWA не конфликтует с ним

### Что реализовано

**Манифест** (`/manifest.json`, отдаётся FastAPI):
- `display: standalone` — открывается без адресной строки браузера
- `theme_color: #111827` — цвет статус-бара совпадает с топбаром
- `background_color: #faf9f6` — бежевый фон splash-экрана совпадает с темой чата
- Иконки: 192×192, 512×512 (Android), 180×180 apple-touch-icon (iOS)

**Service Worker** (`/sw.js`, отдаётся FastAPI с заголовком `Service-Worker-Allowed: /`):
- Cache-first для `/static/*` — CSS, JS, иконки грузятся мгновенно из кэша
- Network-first для `/api/*` — всегда свежий ответ от LLM; при офлайне возвращает JSON `{error: "Нет соединения"}`
- Страницы кэшируются при первом посещении, доступны офлайн (но без ответов от LLM)

**Иконки** (`interfaces/web/static/icons/`):
- Сгенерированы из `logo.png` через Pillow
- Бежевый фон (#faf9f6) + логотип по центру с отступом 18% — iOS и Android сами обрезают в squircle
- Также добавлен `favicon.ico` (16/32/48 px) и `icon-32.png` для вкладки браузера

**Мета-теги в `base.html`**:
- `<link rel="manifest">` — подключает манифест
- `apple-mobile-web-app-capable` — standalone-режим на iOS
- `apple-mobile-web-app-status-bar-style: black-translucent` — статус-бар поверх приложения
- `viewport-fit=cover` — нужен для `env(safe-area-inset-*)` на iPhone с чёлкой

**Мобильный layout (фикс клавиатуры)**:
- `--vh` через Visual Viewport API — высота экрана обновляется при появлении клавиатуры (на iOS `100dvh` не реагирует на клавиатуру)
- `position: fixed` на топбар при `max-width: 1023px` — не уезжает при прокрутке body браузером
- `env(safe-area-inset-bottom)` в padding поля ввода — отступ от домашнего индикатора iPhone
- `overscroll-behavior: none` — отключает «резиновый» скролл body
- Кнопка «Отправить»: `mousedown → preventDefault()` — клавиатура не прячется при нажатии
- `-webkit-tap-highlight-color: transparent` — убрана синяя подсветка при касании

## Последствия

- **Проще:** запуск приложения с домашнего экрана телефона без адресной строки; статика грузится из кэша — быстрее при плохом Wi-Fi
- **Сложнее:** service worker кэширует статику — при обновлении CSS/JS нужно обновить имя кэша `ragv2-v1` в `sw.js`, иначе старая версия останется у пользователей
- **Ограничение:** установка PWA через Chrome требует HTTPS; для локальной сети — использовать ngrok, Tailscale или локальный сертификат через mkcert
- **Следующий шаг:** при переходе на Docker + обратный прокси (nginx) добавить Let's Encrypt — тогда PWA заработает полноценно через HTTPS без костылей
